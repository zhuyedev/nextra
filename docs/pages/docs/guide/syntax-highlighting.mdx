# 语法高亮

Nextra 使用 [Shiki](https://shiki.matsu.io) 在构建时进行语法高亮。这非常可靠且高效。例如，在您的 Markdown 文件中添加以下内容：

````md filename="Markdown"
```js
console.log('hello, world')
```
````

最终效果：

```js
console.log('hello, world')
```

## 特性

### Highlighting Lines

通过在代码块中添加 `{}` 属性，可以高亮特定的代码行：

````md filename="Markdown"
```js {1,4-5}
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```
````

最终效果：

```js {1,4-5}
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

### 高亮子字符串

通过在代码块中添加 `//` 属性，可以高亮特定的子字符串：

````md filename="Markdown"
```js /useState/
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```
````

最终效果：

```js /useState/
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

您可以通过在子字符串后添加数字来只高亮部分出现的子字符串，例如：`/str/1`，或者高亮多个特定的部分：`/str/1-3`、`/str/1,3`。

### 复制按钮

通过添加 `copy` 属性，当用户悬停时，将在代码块上添加一个复制按钮：

````md filename="Markdown"
```js copy
console.log('hello, world')
```
````

最终效果：

```js copy
console.log('hello, world')
```

您可以通过在 Nextra 配置文件（`next.config.js` 文件）中设置 `defaultShowCopyCode: true` 来全局启用此功能。一旦全局启用，您可以通过 `copy=false` 属性来禁用它。

## 动态内容

由于语法高亮是在构建时完成的，您不能在代码块中使用动态内容。然而，由于 MDX 非常强大，可以通过客户端 JS 来解决这个问题。例如：

import { useEffect, useRef } from 'react'

export function DynamicCode({ children }) {
  const ref = useRef()
  const tokenRef = useRef()
  // 从 DOM 中找到对应的标记
  useEffect(() => {
    if (ref.current) {
      const token = [...ref.current.querySelectorAll('code span')].find(
        el => el.innerText === '1'
      )
      tokenRef.current = token
    }
  }, [])
  return (
    <>
      <div ref={ref} style={{ marginTop: '1.5rem' }}>
        {children}
      </div>
      <a
        onClick={() => {
          tokenRef.current.innerText =
            (parseInt(tokenRef.current.innerText) || 0) + 1
        }}
        style={{
          cursor: 'pointer',
          userSelect: 'none'
        }}
      >
        Increase the number
      </a>
      <a
        onClick={() => {
          tokenRef.current.innerText = '1 + 1'
        }}
        style={{
          marginLeft: '.5rem',
          cursor: 'pointer',
          userSelect: 'none'
        }}
      >
        Change to `1 + 1`
      </a>
    </>
  )
}

<DynamicCode>
```js filename="dynamic_code.js"
function hello () {
  const x = 2 + 3
  console.log(1)
}
```
</DynamicCode>

这种方法的限制是更新的内容不会重新高亮显示。例如，如果我们将数字更新为 `1 + 1`，它将被错误地高亮显示。

查看[代码](https://github.com/shuding/nextra/blob/main/docs/pages/docs/guide/syntax-highlighting.mdx)了解其工作原理。

## 禁用语法高亮

您可以选择不使用语法高亮，改用自己的方案。您可以通过在 Nextra 配置文件（`next.config.js` 文件）中设置 `codeHighlight: false` 来全局禁用语法高亮。

## 自定义语法

Shiki 接受[VSCode TextMate 语法](https://code.visualstudio.com/api/language-extensions/syntax-highlight-guide)用于自定义语言语法的语法高亮。

您可以通过覆盖 Nextra 配置中的 `next.config.js` 文件内的 `mdxOptions.rehypePrettyCodeOptions` 选项中的 `getHighlighter` 函数来提供这些语法：

```js filename="next.config.js" {13-18}
import { BUNDLED_LANGUAGES } from 'shiki'

nextra({
  // ... other options
  mdxOptions: {
    rehypePrettyCodeOptions: {
      getHighlighter: options =>
        getHighlighter({
          ...options,
          langs: [
            ...BUNDLED_LANGUAGES,
            // 自定义语法选项，详见 Shiki 文档了解如何提供这些选项
            {
              id: 'my-lang',
              scopeName: 'source.my-lang',
              aliases: ['mylang'], // 除了 id，别名也将包含在允许使用的名称中，用于编写 markdown。
              path: '../../public/syntax/grammar.tmLanguage.json'
            }
          ]
        })
    }
  }
})
```

## 自定义主题

在 `mdxOptions.rehypePrettyCodeOptions` 中，您还可以提供自定义主题，而不是[依赖 CSS 变量](/docs/guide/syntax-highlighting):

```js filename="next.config.js" {4}
nextra({
  // ...其他选项
  mdxOptions: {
    rehypePrettyCodeOptions: {
      // VSCode 主题或内置 Shiki 主题，详见 Shiki 文档了解更多信息
      theme: JSON.parse(
        readFileSync('./public/syntax/arctis_light.json', 'utf8')
      )
    }
  }
})
```

### 多主题

Nextra 目前不支持多主题。因为 Shiki 为每个主题渲染多个代码块，并为它们添加 `data-theme` 属性，例如 `data-theme="dark"`。

<Callout type="info">
  然而未来将会支持多主题。 你可以在 shikiji (Shiki 的新分支) 中跟踪进度。
  https://github.com/antfu/shikiji#multiple-themes
  已经支持多主题而无需渲染多个代码块。
</Callout>
