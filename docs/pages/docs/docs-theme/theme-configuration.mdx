# 主题配置

主题通过 [theme.config.jsx](/docs/docs-theme/start#create-docs-theme-config) 文件进行配置。该文件应导出一个包含您的配置的对象，例如：

```jsx filename="theme.config.jsx"
export default {
  project: {
    link: 'https://github.com/shuding/nextra'
  },
  logo: <b>Project</b>
}
```

## 全局配置

### 文档仓库

设置文档的仓库 URL。它用于生成“[编辑此页面](#edit-link)”链接和“[反馈](#feedback-link)”链接。


<OptionTable

#### 指定路径

如果文档位于一个 monorepo、子文件夹或仓库的不同分支中，您可以将 `docsRepositoryBase` 设置为文档 `pages/` 文件夹的根路径。例如：

```js
export default {
  docsRepositoryBase: 'https://github.com/shuding/nextra/tree/main/docs'
}
```

然后 Nextra 将自动为所有页面生成正确的文件路径。

### SEO 选项

您可以通过 `useNextSeoProps` 选项配置与 SEO 相关的设置。`useNextSeoProps` 函数返回将传递给[Next SEO](https://github.com/garmeeh/next-seo#add-seo-to-page)组件的属性。

它也是一个钩子，因此您可以在其中使用 `useRouter` 等 API 来有条件地返回值。

<OptionTable
  options={[
    [
      'useNextSeoProps',
      '() => NextSeoProps',
      '一个 React 钩子函数，返回 Next SEO 的选项。'
    ]
  ]}
/>

例如，我们可以让所有页面渲染相同的 `<title>` 后缀：

import titleSuffixImage from '../../../public/assets/docs/title-suffix.png'

<Screenshot src={titleSuffixImage} alt="标题后缀" />

```js
export default {
  useNextSeoProps() {
    return {
      titleTemplate: '%s – SWR'
    }
  }
}
```

`%s` 是一个占位符，将被页面标题替换。

您还可以有条件地返回它，以避免在主页上添加后缀：

```js
import { useRouter } from 'next/router'

export default {
  useNextSeoProps() {
    const { asPath } = useRouter()
    if (asPath !== '/') {
      return {
        titleTemplate: '%s – SWR'
      }
    }
  }
}
```

#### 静态 head 标签

如果您只有静态 head 标签，可以直接将它们放入 `head`。例如：

```jsx
export default {
  head: (
    <>
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <meta property="og:title" content="Nextra" />
      <meta property="og:description" content="The next site builder" />
    </>
  )
}
```

#### 基于页面的动态标签

您还可以使用 _函数组件_ 作为 `head`，根据当前页面的 front matter 动态生成头标签。例如：

```jsx
import { useRouter } from 'next/router'
import { useConfig } from 'nextra-theme-docs'

export default {
  head: () => {
    const { asPath, defaultLocale, locale } = useRouter()
    const { frontMatter } = useConfig()
    const url =
      'https://my-app.com' +
      (defaultLocale === locale ? asPath : `/${locale}${asPath}`)

    return (
      <>
        <meta property="og:url" content={url} />
        <meta property="og:title" content={frontMatter.title || 'Nextra'} />
        <meta
          property="og:description"
          content={frontMatter.description || 'The next site builder'}
        />
      </>
    )
  }
}
```

您可以参考 [`useConfig`](/docs/docs-theme/api/use-config) API 部分以获取有关 `useConfig` 钩子和 `frontMatter` 对象的更多信息。

### 菜单折叠层级

默认情况下，侧边栏菜单在层级 `2` 时折叠。您可以通过设置 `sidebar.defaultMenuCollapseLevel` 为不同的数字来更改它。例如，当设置为 `1` 时，每个文件夹默认会折叠，当设置为 `Infinity` 时，所有嵌套文件夹默认会展开。

如果 `sidebar.autoCollapse` 设置为 `true`，那么所有不包含活动/聚焦路由的文件夹将自动折叠到 `sidebar.defaultMenuCollapseLevel` 设置的层级。例如，如果 `defaultMenuCollapseLevel` 是 `2`，那么顶级文件夹将不会自动折叠。

### 选项

```jsx
export default {
  themeSwitch: {
    useOptions() {

        light: '浅色',
        dark: '深色',
        system: '系统'
      }
    }
  }
}
```
